
# خطة إصلاح توليد الجدول للدفعة الواحدة + حفظ الجداول

## المشكلة الحالية

عند توليد جدول لدفعة معينة، يحدث خطأ:
```
duplicate key value violates unique constraint "schedule_entries_room_id_time_slot_id_key"
```

**السبب:** الكود يحذف إدخالات الدفعة المحددة فقط، لكنه لا يأخذ في الاعتبار الإدخالات الموجودة من الدفعات الأخرى عند جدولة القاعات.

---

## الحل المقترح

### الجزء الأول: إصلاح مشكلة التعارض

**تعديل Edge Function** (`supabase/functions/generate-schedule/index.ts`):

| المشكلة | الحل |
|---------|------|
| لا يتم تحميل الإدخالات الموجودة من الدفعات الأخرى | جلب كل `schedule_entries` للمستخدم لمعرفة الفترات المشغولة |
| `occupiedRoomSlots` يبدأ فارغاً | ملء `occupiedRoomSlots` بالإدخالات الموجودة من الدفعات الأخرى |
| `occupiedProfessorSlots` يبدأ فارغاً | ملء `occupiedProfessorSlots` بالإدخالات الموجودة |

**الخطوات:**
1. بعد حذف إدخالات الدفعة المحددة، جلب الإدخالات المتبقية
2. ملء `occupiedRoomSlots` و `occupiedProfessorSlots` و `occupiedGroupSlots` بالبيانات الموجودة
3. هذا يمنع جدولة محاضرة في قاعة مشغولة أو لدكتور لديه محاضرة

```typescript
// بعد حذف إدخالات الدفعة المحددة
// جلب الإدخالات المتبقية للتأكد من عدم التعارض
const { data: existingEntries } = await supabase
  .from('schedule_entries')
  .select('*, subject:subjects(professor_id, group_id)')
  .eq('user_id', user_id);

// ملء الخانات المشغولة
for (const entry of existingEntries || []) {
  occupiedRoomSlots.add(`${entry.room_id}-${entry.time_slot_id}`);
  if (entry.subject?.professor_id) {
    occupiedProfessorSlots.add(`${entry.subject.professor_id}-${entry.time_slot_id}`);
  }
  if (entry.subject?.group_id) {
    occupiedGroupSlots.add(`${entry.subject.group_id}-${entry.time_slot_id}`);
  }
}
```

### الجزء الثاني: إضافة ميزة حفظ الجداول

#### 1. إنشاء جدول جديد `saved_schedules`

| العمود | النوع | الوصف |
|--------|------|-------|
| `id` | uuid | المفتاح الأساسي |
| `user_id` | uuid | صاحب الجدول |
| `name` | text | اسم الجدول |
| `group_id` | uuid | الدفعة (اختياري) |
| `is_active` | boolean | هل هو الجدول النشط |
| `created_at` | timestamp | تاريخ الإنشاء |

#### 2. تعديل جدول `schedule_entries`

إضافة عمود `schedule_id` للربط مع الجدول المحفوظ:
- يتيح حفظ عدة جداول لنفس الدفعة
- يمكن التبديل بين الجداول المحفوظة

#### 3. واجهة المستخدم

**عناصر جديدة في صفحة الجدول:**

```text
┌─────────────────────────────────────────────────────┐
│ [اختر الدفعة ▼]  [الجداول المحفوظة ▼]              │
│                                                     │
│ ┌──────────────────────────────────────────────┐   │
│ │ الجداول المحفوظة:                           │   │
│ │  • جدول الفصل الأول - السبت 15 يناير ✓     │   │
│ │  • جدول تجريبي - الخميس 20 يناير          │   │
│ │  + حفظ الجدول الحالي                        │   │
│ └──────────────────────────────────────────────┘   │
│                                                     │
│ [حفظ باسم جديد]  [توليد]  [مسح]  [تصدير PDF]       │
└─────────────────────────────────────────────────────┘
```

**الوظائف:**
- **حفظ باسم**: فتح نافذة لإدخال اسم الجدول وحفظه
- **عرض المحفوظة**: قائمة بالجداول المحفوظة مع إمكانية:
  - تحميل جدول محفوظ
  - تفعيل جدول كجدول رئيسي
  - حذف جدول محفوظ

## خطوات التنفيذ

1. **قاعدة البيانات:**
   - إنشاء جدول `saved_schedules`
   - إضافة عمود `schedule_id` لجدول `schedule_entries`
   - إضافة سياسات RLS

2. **Edge Function:**
   - جلب الإدخالات الموجودة بعد الحذف
   - ملء الخانات المشغولة قبل بدء الجدولة
   - دعم إنشاء جدول محفوظ جديد

3. **React Hooks:**
   - `useSavedSchedules()` - جلب الجداول المحفوظة
   - `useSaveSchedule()` - حفظ جدول جديد
   - `useLoadSchedule()` - تحميل جدول محفوظ
   - `useDeleteSchedule()` - حذف جدول محفوظ

4. **واجهة المستخدم:**
   - قائمة منسدلة للجداول المحفوظة
   - نافذة حفظ جدول جديد
   - أزرار إدارة الجداول

---

## النتيجة المتوقعة

1. **إصلاح الخطأ:** توليد الجدول لدفعة واحدة يعمل بدون تعارض مع الدفعات الأخرى
2. **حفظ الجداول:** يمكن حفظ عدة نسخ من الجدول بأسماء مختلفة
3. **استعادة سهلة:** التبديل بين الجداول المحفوظة بضغطة واحدة
4. **تنظيم أفضل:** كل دفعة يمكن أن يكون لها عدة جداول محفوظة
