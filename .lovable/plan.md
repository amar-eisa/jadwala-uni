
# خطة استبدال شاشة التقدم بإحصائيات النظام

## الفكرة
عند اكتمال إعداد النظام (جميع الخطوات الخمس مكتملة)، سيتم إخفاء بطاقة "تقدم إعداد النظام" واستبدالها ببطاقة إحصائيات متقدمة تعرض معلومات مفيدة عن النظام.

---

## التغييرات المطلوبة

### تعديل ملف `src/pages/Dashboard.tsx`

#### 1. إضافة منطق التحقق من الاكتمال
استخدام المتغير `progressPercentage === 100` للتحقق من اكتمال الإعداد.

#### 2. عرض مشروط للبطاقات

```text
إذا كانت progressPercentage < 100:
    ├─ عرض بطاقة "تقدم إعداد النظام" (الحالية)
    
إذا كانت progressPercentage === 100:
    ├─ عرض بطاقة "إحصائيات النظام" (جديدة)
```

#### 3. بطاقة الإحصائيات الجديدة ستتضمن:
- **إجمالي المحاضرات**: عدد schedule_entries
- **عدد الأيام المجدولة**: أيام فريدة من time_slots
- **متوسط المحاضرات لكل يوم**: إجمالي المحاضرات / عدد الأيام
- **نسبة استخدام القاعات**: محاضرات / (قاعات × فترات)
- **حالة النظام**: جاهز للاستخدام

## تفاصيل التنفيذ

### الكود المقترح للإحصائيات:

```typescript
// حساب إحصائيات النظام
const isSetupComplete = completedSteps === totalSteps;

// أيام فريدة من الفترات الزمنية
const uniqueDays = [...new Set(timeSlots?.map(ts => ts.day) || [])];
const totalDays = uniqueDays.length;

// متوسط المحاضرات لكل يوم
const avgLecturesPerDay = totalDays > 0 
  ? ((scheduleEntries?.length || 0) / totalDays).toFixed(1) 
  : 0;

// نسبة استخدام القاعات (تقريبية)
const totalSlots = (rooms?.length || 1) * (timeSlots?.length || 1);
const utilizationRate = totalSlots > 0 
  ? Math.round(((scheduleEntries?.length || 0) / totalSlots) * 100) 
  : 0;
```
