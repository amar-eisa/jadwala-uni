
# خطة نظام اعتماد المستخدمين وإدارة الاشتراكات

## ملخص المتطلبات

1. **الحسابات الجديدة في حالة انتظار** - عند إنشاء حساب جديد، يكون في حالة "pending" حتى يعتمده المدير
2. **شاشة اعتماد المستخدمين** - إضافة قسم للمدير لاعتماد أو رفض المستخدمين الجدد
3. **حذف المستخدمين** - إمكانية حذف أي مستخدم من النظام
4. **فترة الاشتراك** - تحديد تاريخ بداية ونهاية الاشتراك لكل مستخدم

---

## التغييرات المطلوبة

### 1. تغييرات قاعدة البيانات

#### أ. تعديل Trigger إنشاء الاشتراك
تغيير الحالة الافتراضية من `'active'` إلى `'pending'`:

```sql
CREATE OR REPLACE FUNCTION public.handle_new_user_subscription()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.subscriptions (user_id, plan_name, status, price)
  VALUES (new.id, 'free', 'pending', 0);
  RETURN new;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER SET search_path = public;
```

#### ب. إضافة سياسة حذف للمدراء على جدول profiles

```sql
CREATE POLICY "Admins can delete profiles" ON public.profiles
  FOR DELETE USING (has_role(auth.uid(), 'admin'));
```

#### ج. إضافة سياسات حذف للمدراء على جداول البيانات المرتبطة
لضمان حذف جميع بيانات المستخدم عند حذفه.

### 2. تحديث واجهة المستخدم

#### أ. تعديل AuthContext.tsx
- تعديل رسالة النجاح لتوضح أن الحساب في انتظار الاعتماد
- إضافة رسالة توضيحية للمستخدم

#### ب. تعديل AuthPage.tsx
- عرض رسالة واضحة للمستخدمين الجدد بأن حسابهم في انتظار موافقة المدير
- منع الدخول للحسابات غير المعتمدة

#### ج. تعديل ProtectedRoute.tsx
- إضافة فحص لحالة الاشتراك `pending`
- توجيه المستخدمين غير المعتمدين لصفحة انتظار

#### د. تحديث UserManagement.tsx الرئيسي
سيتم إضافة:
- **قسم المستخدمين المعلقين**: عرض المستخدمين في انتظار الاعتماد
- **أزرار الاعتماد/الرفض**: لكل مستخدم معلق
- **زر الحذف**: لحذف أي مستخدم
- **تحديد فترة الاشتراك**: اختيار تاريخ البداية والنهاية

---

### 3. تحديث Hooks

#### أ. تعديل useAdmin.ts
إضافة الوظائف التالية:

```typescript

// رفض مستخدم (حذف الحساب)

// تحديث فترة الاشتراك

```

#### ب. تعديل useSubscription.ts
- إضافة فحص حالة `pending` للاشتراك
- إضافة hook للتحقق من حالة الاعتماد

---

### 4. إنشاء صفحة الانتظار

#### صفحة PendingApprovalPage.tsx
صفحة بسيطة تُعرض للمستخدمين الذين تم إنشاء حسابهم ولكن في انتظار الاعتماد:

- رسالة توضيحية بأن الحساب قيد المراجعة
- زر لتسجيل الخروج
- معلومات التواصل (اختياري)

## مسار المستخدم بعد التعديل

```text
1. مستخدم جديد يسجل حساب
         ↓
2. يظهر له رسالة "حسابك في انتظار موافقة المدير"
         ↓
3. يتم تحويله لصفحة الانتظار (لا يستطيع الوصول للتطبيق)
         ↓
4. المدير يرى المستخدم في قسم "في الانتظار"
         ↓
5. المدير يختار:
   - [اعتماد] → الحساب يصبح نشط + يحدد فترة الاشتراك
   - [رفض] → يتم حذف الحساب
         ↓
6. المستخدم المعتمد يستطيع الآن استخدام التطبيق
```

## ملاحظات أمنية مهمة

1. **حذف المستخدم**: سيتم حذف جميع البيانات المرتبطة (القاعات، الأساتذة، المواد، إلخ)
2. **التحقق من الصلاحيات**: جميع العمليات تتم عبر RLS policies
3. **لا يمكن للمدير حذف نفسه**: سيتم إضافة فحص لمنع ذلك
