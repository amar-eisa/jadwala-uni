
# خطة إضافة بوابة الطلاب

## الفكرة العامة
إنشاء قسم منفصل للطلاب على رابط `/student` يمكنهم من خلاله:
- التسجيل وتسجيل الدخول بحساب طالب
- اختيار مجموعتهم الدراسية
- عرض الجدول الأسبوعي الخاص بمجموعتهم (الجدول النشط فقط - للقراءة فقط)

## التغييرات المطلوبة

### 1. قاعدة البيانات

**جدول جديد: `student_profiles`**
- `id` (uuid, primary key)
- `user_id` (uuid, references auth.users, unique)
- `full_name` (text)
- `email` (text)
- `student_id_number` (text) - رقم القيد الجامعي
- `group_id` (uuid, nullable) - المجموعة المختارة
- `created_at`, `updated_at`

**دور جديد:** إضافة `'student'` إلى `app_role` enum

**سياسات الأمان (RLS):**
- الطالب يمكنه رؤية وتعديل ملفه الشخصي فقط
- الطالب يمكنه قراءة المجموعات (`student_groups`) والفترات الزمنية (`time_slots`) والجداول النشطة فقط
- المدير يمكنه رؤية جميع الطلاب

**سياسات إضافية على الجداول الحالية:**
- إضافة سياسة SELECT على `schedule_entries` و `saved_schedules` و `time_slots` و `rooms` و `subjects` و `professors` و `student_groups` تسمح للطلاب بقراءة البيانات المتعلقة بمجموعتهم

### 2. Trigger لتسجيل الطلاب
- عند تسجيل مستخدم جديد بدور `student`، يتم إنشاء سجل في `student_profiles` تلقائيا

### 3. صفحات جديدة

**`/student/auth` - صفحة تسجيل دخول الطلاب**
- نموذج تسجيل يطلب: الاسم، البريد الإلكتروني، كلمة المرور، رقم القيد
- نموذج تسجيل الدخول
- تصميم مميز عن صفحة الإدارة لتمييز بوابة الطلاب

**`/student` - لوحة تحكم الطالب**
- اختيار المجموعة الدراسية (إذا لم يختر بعد)
- عرض الجدول الأسبوعي للمجموعة المختارة (قراءة فقط)
- تصميم بسيط ونظيف بنفس أسلوب التطبيق الزجاجي

**`/student/profile` - صفحة الملف الشخصي**
- تعديل البيانات الشخصية وتغيير المجموعة

### 4. مكونات جديدة

- `StudentProtectedRoute` - حماية صفحات الطلاب (يتحقق من دور `student`)
- `StudentLayout` - تخطيط بسيط للطلاب بدون الشريط الجانبي الكامل (شريط علوي فقط)
- `StudentTimetableView` - عرض الجدول للقراءة فقط (بدون سحب/إفلات)

### 5. تحديثات على الملفات الحالية

- **`src/App.tsx`**: إضافة مسارات `/student`, `/student/auth`, `/student/profile`
- **`src/contexts/AuthContext.tsx`**: إضافة دالة `signUpStudent` لتسجيل الطلاب مع دور `student`

## التفاصيل التقنية

### هيكل الملفات الجديدة
```text
src/
  pages/
    student/
      StudentAuthPage.tsx
      StudentDashboard.tsx
      StudentProfilePage.tsx
  components/
    student/
      StudentProtectedRoute.tsx
      StudentLayout.tsx
      StudentTimetableView.tsx
  hooks/
    useStudentProfile.ts
    useStudentSchedule.ts
```

### تدفق عمل الطالب
```text
1. الطالب يزور /student/auth
2. يسجل حسابا جديدا (اسم + بريد + كلمة مرور + رقم قيد)
3. يتم إنشاء حساب بدور "student"
4. بعد تسجيل الدخول -> /student
5. يختار مجموعته الدراسية
6. يرى الجدول الأسبوعي لمجموعته
```

### سياسات RLS للطلاب
- الطلاب يقرأون فقط الجداول النشطة (`is_active = true`)
- الطلاب يرون فقط محاضرات مجموعتهم المحددة
- لا يمكن للطلاب تعديل أو حذف أي بيانات في الجداول الإدارية

### ملاحظات أمنية
- فصل كامل بين بوابة الطلاب وبوابة الإدارة
- الطلاب لا يمكنهم الوصول إلى صفحات الإدارة والعكس صحيح
- كل البيانات المعروضة للطلاب هي للقراءة فقط
