

# خطة إعادة تصميم شاشة توليد الجدول

## تحليل المشاكل الحالية

### المشاكل في التصميم الحالي:

1. **صفحة مزدحمة بالوظائف**: الشاشة الحالية تجمع بين توليد الجدول وعرضه في نفس المكان مما يجعلها معقدة
2. **تداخل المفاهيم**: عند الضغط على "توليد مرة أخرى" يتم استبدال الجدول الحالي مباشرة دون إنشاء نسخة مختلفة
3. **عرض الجداول المحفوظة محدود**: القائمة المنسدلة `SavedSchedulesMenu` لا تعرض معلومات كافية (رقم الإصدار، الدفعة، إلخ)
4. **لا يوجد قسم منفصل للأرشيف**: المستخدم لا يستطيع استعراض كل الجداول السابقة بشكل مريح
5. **التوليد لا ينتج نتائج مختلفة**: الخوارزمية ثابتة ولا تستخدم عشوائية لإنتاج جداول متنوعة

## التغييرات المطلوبة

### 1. تعديل Edge Function لإنتاج جداول مختلفة

| الملف | التغيير |
|-------|---------|
| `supabase/functions/generate-schedule/index.ts` | إضافة عشوائية للخوارزمية |

**التفاصيل:**

```typescript
// إضافة خلط عشوائي للمواد قبل الجدولة
function shuffleArray<T>(array: T[]): T[] {
  const shuffled = [...array];
  for (let i = shuffled.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
  }
  return shuffled;
}

// في دالة التوليد:
const sessionQueue = shuffleArray([...allSessions]);  // خلط المحاضرات
const sortedDays = shuffleArray([...days]);           // خلط الأيام
```

**النتيجة:** كل ضغطة على "توليد" ستنتج جدولاً مختلفاً.

### 3. إضافة رقم الإصدار في قاعدة البيانات

| التغيير | التفاصيل |
|---------|----------|
| Migration | إضافة عمود `version` لجدول `saved_schedules` |

```sql
ALTER TABLE saved_schedules 
ADD COLUMN version integer NOT NULL DEFAULT 1;
```

**منطق حساب الإصدار:**
- عند حفظ جدول جديد لدفعة معينة: نحسب عدد الجداول الموجودة لهذه الدفعة + 1

### 4. إعادة تصميم TimetablePage

| القسم | التغيير |
|-------|---------|
| Header | تبسيط - عنوان + وصف فقط |
| Generator Card | قسم بسيط لاختيار الدفعة + زر توليد |
| Saved Schedules Card | قسم منفصل بـ cards لكل جدول محفوظ |
| Timetable View | يبقى كما هو مع تحسينات بسيطة |

**تدفق العمل الجديد:**

```
1. المستخدم يختار الدفعة
   ↓
2. يضغط "توليد جدول جديد"
   ↓
3. النظام يولد جدولاً ويعرضه مباشرة (كـ "مسودة")
   ↓
4. المستخدم يعاين الجدول
   ↓
5. إذا أعجبه: يضغط "حفظ" → يُحفظ كإصدار جديد
   إذا لم يعجبه: يضغط "توليد مرة أخرى" → جدول جديد مختلف
   ↓
6. الجداول المحفوظة تظهر في قسم "الجداول المحفوظة"
   يمكن التبديل بينها في أي وقت
```

### 5. تحديث hook الحفظ لإضافة الإصدار

| الملف | التغيير |
|-------|---------|
| `src/hooks/useSavedSchedules.ts` | حساب الإصدار تلقائياً |

```typescript
// عند الحفظ:
const existingVersions = await supabase
  .from('saved_schedules')
  .select('version')
  .eq('user_id', user.id)
  .eq('group_id', groupId || null)
  .order('version', { ascending: false })
  .limit(1);

const nextVersion = (existingVersions.data?.[0]?.version || 0) + 1;
```

## التحسينات الإضافية

### (أ) مؤشر بصري للمسودة
- عند توليد جدول جديد ولم يُحفظ بعد: عرض badge "مسودة - غير محفوظ"
- لون مختلف للتمييز

### (ب) تحسين تجربة التوليد
- عند الضغط على "توليد":
  1. عرض spinner
  2. عرض الجدول الجديد فوراً
  3. عرض رسالة: "تم إنشاء جدول جديد (غير محفوظ)"

### (ج) إضافة زر "توليد مرة أخرى"
- يظهر بعد أول توليد
- يوضح للمستخدم أن كل ضغطة ستنتج جدولاً مختلفاً
