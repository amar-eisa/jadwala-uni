

## تحسينات واجهة المستخدم - لوحة الإدارة

### 1. بحث وتصفية في قائمة المستخدمين

**في `UserManagement.tsx`:**
- اضافة شريط بحث (Input) اعلى كل تبويب يبحث في الاسم والبريد ورقم الهاتف
- اضافة قائمة تصفية حسب حالة الاشتراك (نشط، في الانتظار، منتهي، الكل)
- اضافة تصفية حسب الدور (مدير، مستخدم، الكل)
- تطبيق الفلترة على `pendingUsers` و `allUsers` باستخدام `useMemo`

### 2. إشعارات فورية عند تسجيل مستخدم جديد

**تغييرات قاعدة البيانات:**
- تفعيل Realtime على جدول `profiles` عبر migration: `ALTER PUBLICATION supabase_realtime ADD TABLE public.profiles;`

**في `UserManagement.tsx`:**
- اضافة اشتراك Realtime على جدول `profiles` يستمع لحدث `INSERT`
- عند وصول مستخدم جديد يظهر toast اشعار بـ sonner يحتوي اسم المستخدم
- يتم تحديث قائمة المستخدمين تلقائيا عبر `invalidateQueries`

### 3. تحسين تجربة الموبايل

**في `UserManagement.tsx`:**
- تحويل جدول "جميع المستخدمين" الى بطاقات (cards) على الشاشات الصغيرة باستخدام responsive classes
- على الموبايل: كل مستخدم يظهر كبطاقة تحتوي الاسم والبريد والهاتف والحالة وازرار الاجراءات
- على سطح المكتب: يبقى الجدول كما هو
- استخدام `useIsMobile` hook الموجود للتبديل بين العرضين

**في `AdminPage.tsx`:**
- تصغير حجم العنوان على الموبايل
- جعل TabsList يتمدد بشكل كامل على الموبايل

### التفاصيل التقنية

**هيكل البحث والتصفية:**
```text
┌─────────────────────────────────────┐
│ 🔍 بحث بالاسم او البريد او الهاتف  │
├──────────┬──────────────────────────┤
│ الحالة ▾ │ الدور ▾                  │
└──────────┴──────────────────────────┘
```

**Realtime subscription:**
- يتم الاشتراك عند mount المكون ويلغى عند unmount
- يستخدم `supabase.channel('new-users').on('postgres_changes', ...)`

**عرض الموبايل للمستخدمين:**
```text
┌────────────────────────┐
│ 👤 اسم المستخدم        │
│ email@example.com      │
│ +249123456789          │
│ ┌──────┐ ┌──────┐      │
│ │ نشط  │ │ متميز│      │
│ └──────┘ └──────┘      │
│ [تعديل] [فاتورة] [حذف]│
└────────────────────────┘
```
